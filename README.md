## ğŸŒŸ The Business Problem

The logistics and freight industry suffers from massive fragmentation. Delivery companies traditionally use separate tools for GPS tracking, driver management, financial accounting, and vehicle maintenance. This results in:
1. **Inefficient Dispatching**: Empty miles and underutilized cargo space.
2. **Hidden Expenses**: Unpredictable vehicle downtime and volatile fuel costs that destroy profit margins.
3. **Data Silos**: Poor communication between the Dispatcher, Safety Officer, and Fleet Manager, leading to delayed decision-making.

## ğŸ’¡ The FleetFlow Solution (Business Impact)

FleetFlow is a dynamic, all-in-one logistics solution designed to eliminate data silos and automate manual workflows. By bringing all departments into a single source of truth, FleetFlow ensures:

* **Maximized ROI**: Predictive analytics and strict fuel logging help companies reduce operational overhead by up to 25%.
* **Proactive Maintenance**: Tracking vehicle lifecycle and repair costs prevents costly breakdowns and increases asset longevity.
* **Driver Safety & Compliance**: CDL license expiration tracking and dynamic safety scoring reduces liability and insurance premiums.
* **Real-time Routing**: Dispatchers visually assign drivers to vehicles based on cargo weight capacity, instantly moving freight without phone calls or paper trails.

## ğŸš€ Getting Started Locally

To get a local development environment up and running, follow these steps:

1. Clone the repository
   ```sh
   git clone https://github.com/Parth-Ramanujj/Fleet_Flow-And-Logistics-Management-System.git
   ```
2. Install client dependencies
   ```sh
   npm install
   ```
3. Setup your MySQL Database schema using the provided raw dumps.
4. Create a `.env` file in the root directory:
   ```env
   DB_HOST=localhost
   DB_USER=root
   DB_PASSWORD=your_mysql_password
   DB_NAME=fleetflow_db
   PORT=5000
   ```
5. Start the frontend client
   ```sh
   npm run dev
   ```
